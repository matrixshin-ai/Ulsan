# 울산 Vault 운영 규칙

## 명령어

### "ingest"
1. vault 내 아직 처리되지 않은 파일을 모두 읽는다 (`wiki/` 디렉토리는 제외).
2. 핵심 엔티티(인물·기관·사업 등)와 개념을 추출한다.
3. 한국어 요약 페이지를 `wiki/`에 작성한다.
4. `wiki/_index.md`를 갱신한다.

### "lint"
`wiki/`를 점검하고 다음을 보고한다 (수정하지 않고 보고만 한다):
- 깨진 위키링크
- 고아 페이지 (어디서도 링크되지 않는 페이지)
- 내용 간 모순
- 내용 공백 (다뤄야 하는데 빠진 주제)

### "query [질문]"
1. 먼저 `wiki/_index.md`를 읽는다.
2. `wiki/` 내용에만 근거해 한국어로 답변한다. wiki에 없는 내용은 추측하지 않는다.

## 제약

- `daily/` 안의 파일은 절대 수정하지 않는다.
- 모든 wiki 페이지는 한국어로 작성한다.
- 모든 wiki 페이지는 다음 frontmatter를 포함해야 한다:
  ```yaml
  type:
  date_updated:
  source_count:
  ```
